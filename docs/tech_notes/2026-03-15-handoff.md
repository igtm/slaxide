# Slaxide 技術引継書

更新日: 2026-03-16

## 1. 現在の状態

このリポジトリは Rust workspace として初期化済み。現時点の構成は次の通り。

- `crates/slaxide-core`
  - タイムラインモデル
  - ルールベース順位付け
  - 設定スキーマ
  - 内蔵テーマ定義
- `crates/slaxide-store`
  - SQLite 永続化
  - store actor ハンドル
- `crates/slaxide-slack`
  - OAuth URL 生成
  - token refresh
  - conversation list / history fetch
  - Socket Mode 接続 bootstrap
  - thread reply API
  - external upload API のラッパ
- `crates/slaxide-platform`
  - notify-rust ベース通知
  - keyring ベース secret store 抽象
- `crates/slaxide-linux-gtk`
  - Linux UI の最初の shell
  - XDG path bootstrap
  - SQLite/store actor からの初期ロード
  - `config.toml` overlay
  - Slack OAuth button / status UI
  - browser redirect URL paste-back flow
  - 初回 Slack history ingest
  - known channel cache と multi-room 切替
  - keyring への token 保存
  - TweetDeck 風 multi-column timeline / thread / settings / admin
  - theme switch
  - channel post / thread reply / share / copy
  - own message edit / delete
  - admin menu から channel create / rename / archive / invite / kick
  - configurable keyboard shortcuts
  - message reaction の quick add / custom add UI
  - live reaction / typing indicator
  - empty state UI
  - theme / cached timeline の SQLite 永続化

重要:

- GTK frontend の sample data 依存は外した
- 起動時に settings / timeline は SQLite から読む
- キャッシュ未存在時は sample ではなく空状態 UI を出す
- Slack OAuth は env var 設定があればブラウザ認証まで動く
- OAuth token は keyring に保存し、期限が近ければ起動時 refresh を試みる
- OAuth 後に公開 / 非公開チャンネルの直近履歴を初回 ingest する
- watched channel が未設定なら ingest 結果から最低限 seed して `Recent` で見えるようにする
- search profile / keyword profile / channel profile / author profile を TOML overlay と UI filter から使える
- channel permission は `read_write` / `read_only` / `hidden` を持ち、composer は writable channel だけを出す
- reaction bar は既存 emoji chip に加えて `+` popover から任意 shortcode / Unicode emoji を追加できる
- live reaction 反映は `reaction_added` / `reaction_removed` の workspace event と `reactions:read` scope が必要
- `reaction_added` / `reaction_removed` が来ない workspace でも、`message_changed` で reactions が乗るケースは live snapshot から追従する
- typing indicator は `user_typing` workspace event を追加すると channel ごとに表示される
- timeline の copy icon 右に share icon があり、`chat.getPermalink` で Slack message URL を clipboard にコピーできる
- 本文中の Slack message permalink は、cache 内に対象 message があれば compact preview として展開される
- live sync / Socket Mode は実装済みで、desktop notification と timeline rerank に接続済み
- settings には Slack credential fallback と shortcut summary があり、`.env` / exported env が保存値より優先される
- default scope set は local feature 前提で `chat:write` / `reactions:*` / `users:read` / `channels:write*` / `groups:write*` を含む
- ただし event 設定や app token など Slack app 側の設定は環境ごとに確認が要る

確認済みコマンド:


2026-03-15 時点で通過済み:

- `slaxide-linux-gtk`

未確認:

- 実 Slack 接続
- 実機 Linux GUI の長時間手動確認

補足:

- workspace の `rust-version` は `1.92` に下げた
- Hyprland では GTK renderer を `cairo` に寄せると起動する
- `slaxide-linux-gtk` は Hyprland かつ `GSK_RENDERER` 未指定時に `cairo` を自動設定する
- Slack OAuth の env var:
  - `SLAXIDE_SLACK_CLIENT_ID`
  - `SLAXIDE_SLACK_CLIENT_SECRET`
  - 任意で `SLAXIDE_SLACK_REDIRECT_URI`
  - 任意で `SLAXIDE_SLACK_USER_SCOPES`
- 開発中は `.env` も読み、`.env` / exported env が Settings 保存値より優先される
- sandbox 内では `/dev/dri/*` に触れられないため GPU renderer の確認は未完了
- Slack app 側の Redirect URL は `https://...` 必須になっている
- custom URL scheme の deeplink は Slack OAuth redirect に使えない
- 現在の desktop flow は `https://127.0.0.1/slack/callback` を既定にして、ブラウザで redirect された URL を app に貼り戻す
- config overlay を使うなら `~/.config/slaxide/config.toml` に置く。サンプルは `docs/config.example.toml`

## 4. Manjaro 再起動後の最初の作業

Manjaro 上でまだ system dependency を入れていない場合は先に入れる。

```bash
sudo pacman -Syu --needed base-devel pkgconf gtk4 graphene libadwaita
```

その後、最低限この順番で確認する。

```bash
pkg-config --modversion gtk4
pkg-config --modversion graphene-gobject-1.0
cargo test
cargo test -p slaxide-linux-gtk
cargo check -p slaxide-linux-gtk
cargo run -p slaxide-linux-gtk
```

Slack OAuth を試すときの最小設定:

```bash
export SLAXIDE_SLACK_CLIENT_ID=...
export SLAXIDE_SLACK_CLIENT_SECRET=...
export SLAXIDE_SLACK_REDIRECT_URI=https://127.0.0.1/slack/callback
cargo run -p slaxide-linux-gtk
```

Slack app の OAuth & Permissions にも同じ `https://127.0.0.1/slack/callback` を Redirect URL として登録する。
認可後はブラウザが接続失敗画面になるが、address bar の redirect URL 全体を Slaxide に貼り戻せばよい。

## 9. 注意点

- `Cargo.toml` の default-members から `slaxide-linux-gtk` は外してある。理由は system dependency が無い環境でも backend の `cargo test` を回せるようにするため。
- 現在の UI はプロトタイプであり、表示は出る前提だが本番データフローは未接続。
- keyring 実装は `use_native_store(true)` を呼ぶので、Linux では keyutils ではなく Secret Service 寄りの挙動を期待している。セッション側の keyring 解錠状態は確認すること。
- libadwaita を深く使い始めるとテーマ自由度と競合しやすい。Zen テーマ優先なら CSS 適用戦略を早めに見直すこと。
